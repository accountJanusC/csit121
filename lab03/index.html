My first HTML page

What I Found Out

  1. The first tidbit of unfamiliar material I was introduced to was bolding and italicising the text. I learned there were differences between the <b> and <strong> tags as well as the <i> and <em> tags from a W3docs URL in the 'Structuring Text' section of Module 02. The W3docs article did not explain the differences in detail so I looked up other sources from the Stack Overflow and FreeCodecamp websites. I found out the <strong> and <em> tags should be used to convey actual emphasis which would be noticed in web browsers and search engines. They would impact how the the text would be read out for the visually impaired while the <b> and <i> tags have no such effect.
  2. I also learned from a CSIT 121 lecture about the tag for adding line breaks.
    For line breaks like the one here for example, if I want to shorten the amount of text in one line, simply pressing the Enter key is not going to do it. The 'Structuring Text' section in Module 02 notes that <br> is an empty tag that must be used in another element.
  3. A third aspect of HTML I familiarised myself with was adding hidden comments. I found out how to do it from the HTML Tags Reference page on the W3Schools website. The URL of the Tags Reference page was posted at the top of Module 02's 'Structuring Text' section.

Finding out about the proper way to use headings as well as the HTML format involving tags, attributes, and elements. I found out in the 'HTML Basics Notes' section of CSIT 121's Module 02 that the always lowercase opening tags come first, followed by the attributes, and then the elements. I still didn't quite understand attributes so the W3Schools tutorial section on it really helped explain its definition and layout.

I was initially confused why the 'Structuring Text' section said not to use headings just for sizing, and reading a source from Mozilla Developers Network (MDN) helped me understand. The MDN article advised against using more than one <h1> element on the same page but others up to <h6> are allowed as long as they fit the page's information hierarchy. I've also learned that <h7> and above do not have meaning in HTML. Here are the webpages below which helped me understand how to format attributes and headings properly.

  • HTML Section Heading elements by MDN
  • HTML Attributes by W3Schools

Adding CSS

  1. In Module 03, we learned about CSS aka Cascading Style Sheets. And this assignment allowed me to further practice curly bracket and colon syntax for CSS.
  2. The Module and lectures on this topic also gave me knowledge on CSS formatting, notably the use of the 'link rel' combo to add a style sheet to an HTML page. In particular, the 'link' element should specify a file or resource while the 'rel' attribute should specify known values/keywords such as stylesheet, search, policy, alternate etc.
  3. However, I needed to find resources on my own to better understand how to use ID and class selectors since they were not covered in detail in both Module 02 and 03. I particularly needed to relearn the use of "#" before the ID selector and "." before the class selector.
